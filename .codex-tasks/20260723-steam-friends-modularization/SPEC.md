# Steam 好友模块拆分

## 目标

将好友功能从单层包和近千行页面文件拆分为职责清晰的独立模块，保持全部功能、界面、缓存、导航和测试行为不变。

## 模块结构

```text
steam/friends/
├── domain/          模型、关系语义、筛选规则、网关接口
├── data/            Steam 网络适配器与缓存适配器
├── presentation/    页面状态与异步请求协调
└── ui/              页面装配、列表、详情、卡片及通用视觉元素
```

## 架构要求

1. Activity 仅依赖好友页公开入口。
2. UI 仅通过状态管理模块和领域模型获取行为。
3. 网络与 SharedPreferences 实现位于 data 包。
4. 页面装配、列表、详情、卡片和头像加载分别存放。
5. 保持现有资源名称，避免无关改动。
6. 禁止构建 APK，仅运行测试与 Kotlin 编译。

## 验收标准

原始 `steam/friends` 根包不再堆放实现文件；页面装配文件控制在合理规模；所有引用迁移完成；151 个以上测试套件全部通过；Kotlin 编译通过；提交并推送 main。
